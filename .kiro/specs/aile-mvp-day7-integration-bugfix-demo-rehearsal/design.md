# 设计文档：艾乐学伴 MVP Day 7 — 集成测试、Bug 修复与演示彩排

## 概述

Day 7 目标是在 Day 6 演示资产就绪基础上，完成“可发布演示版本”的最后闭环：

- 端到端集成验证（优先 Docker Compose 冷启动路径）
- 阻塞性与高风险缺陷清零
- 稳定性兜底能力补齐（超时、白屏、关键加载态）
- 演示彩排与备份材料收口

交付标准对齐冲刺计划：可在新环境快速启动并在 15-20 分钟内稳定演示 3 个场景。

## 设计输入与约束

本设计基于以下输入：

- `艾乐学伴MVP(Demo版)-7天冲刺计划.md` 的 Day 7 范围、节奏与验收口径
- `1-艾乐学伴MVP数据库详细设计书.md` 的表结构约束、状态机和查询视图
- `2-艾乐学伴MVP API接口规范全集 (OpenAPI 3.0).md` 的全局错误码、状态枚举、接口契约
- `3-艾乐学伴MVP后端微服务详细设计书.md` 的集成测试建议（API+DB+Redis）与健康检查口径
- `4-艾乐学伴MVP前端应用详细设计书.md` 的前端测试策略与关键旅程
- 现状输入：非 Docker 路径已整体跑通；WSL 与 Docker 已安装并可启动

关键约束：

- 不改动核心业务语义：任务状态机与接口契约保持兼容
- Day 7 以“稳定上线演示”为目标，优先修复阻塞与高风险问题，不做新增大功能
- 回归顺序采用“先非 Docker 基线自检，再 Docker 冷启动验收”，定位环境问题与业务问题

## 范围与边界

### 本次实现范围

- 建立 Day 7 集成测试清单与执行顺序（环境、接口、旅程、异常）
- 实施并记录异常场景测试（断网、AI 慢响应/超时、空数据）
- 增强系统韧性：
  - AI 超时兜底提示（10 秒阈值）
  - 前端 ErrorBoundary 防白屏
  - 核心异步区域 loading/error/empty 三态一致化
- 缺陷治理：按 P0/P1/P2 分级修复并回归
- 演示彩排三轮与演示材料收口（README、一页说明、备用视频指引）

### 明确不在 Day 7 范围

- 新增业务模块或重构微服务边界
- 引入高成本基础设施（如新消息队列、新观测平台）
- 变更数据库核心模型或 OpenAPI 契约定义

## 方案设计

### 1) 集成测试执行架构

采用“两条路径、一套用例”：

- 路径 A（基线）：非 Docker 本地流程回归，确认业务链路无回退
- 路径 B（目标）：Docker Compose 冷启动回归，确认可迁移演示环境

统一测试清单按层次执行：

1. 环境与健康：容器状态、`/health`、前端可访问
2. 核心接口：认证、计划、任务、对话、内容、诊断关键接口
3. 关键旅程：
   - 场景 A：首次使用
   - 场景 B：日清对话 -> 任务生成 -> 执行
   - 场景 C：诊断报告 -> 巩固计划
4. 异常旅程：网络中断、AI 超时、空数据

### 2) 异常场景验证矩阵

- 网络断开：
  - 前端请求失败后显示可理解错误提示与重试入口
  - 页面保持可操作，不出现无限 loading
- AI 超时：
  - 到达阈值后提示“艾乐正在思考，请稍候...”
  - 允许继续等待或重试，不破坏当前会话上下文
- 空数据：
  - 首页任务、执行内容、诊断报告在空结果时显示统一空态
  - 不出现空白容器或控制台报错

### 3) 稳定性增强策略

后端侧：

- 对外部 AI 调用设置明确超时与失败映射，超时统一转为 `503 SERVICE_UNAVAILABLE` 语义
- 对关键异常打点日志（请求链路 ID、失败类型、耗时）

前端侧：

- 在应用根节点引入 ErrorBoundary，兜底渲染降级页面，避免白屏
- 对核心异步区（首页、对话、执行、诊断）统一 loading/error/empty 处理
- 对 `401/403/503` 等关键错误码按 API 规范做明确 UX 反馈

### 4) Bug 修复与回归闭环

缺陷分级标准：

- P0：阻塞演示主路径（必须当日清零）
- P1：影响体验但可绕过（尽量清零）
- P2：低风险体验问题（记录并评估）

处理流程：

1. 记录复现步骤、影响范围、优先级
2. 最小改动修复，避免 Day 7 引入大规模重构
3. 修复后执行“问题点回归 + 主路径冒烟”
4. 更新缺陷清单与回归结果

### 5) 演示彩排设计

按照冲刺计划进行三轮彩排：

- 第 1 轮：严格按脚本走，收集卡点、耗时点、风险点
- 第 2 轮：修复卡点并优化讲述节奏与切换衔接
- 第 3 轮：完整模拟正式演示并计时（目标 15-20 分钟）

彩排输出：

- 最终演示脚本（含每步预期结果与异常应对口径）
- 备用路径（网络异常时切 Demo_Mode 或备用视频）

### 6) 文档与交付物收口

- `README.md` 补齐：启动步骤（Docker/非 Docker）、演示账号、常见问题
- 演示辅助材料：
  - 一页架构与入口说明（可转 PPT）
  - 备用演示视频录制指引与触发条件

## 验收映射（Day 7）

- 集成通过：Docker 冷启动后，关键旅程与核心接口可跑通
- 稳定性达标：AI 超时提示、ErrorBoundary、关键 loading 状态均可验证
- 缺陷可控：P0 清零，P1 明显收敛且有规避方案
- 演示可交付：三轮彩排完成，15-20 分钟内稳定走完三场景
