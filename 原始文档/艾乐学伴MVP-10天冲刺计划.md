# 艾乐学伴 MVP Demo — 10天从容版工作计划

## 与7天版本的核心差异

| 对比项 | 7天冲刺版 | 10天从容版 |
|--------|-----------|------------|
| AI对话 | 基础文字对话 | 多轮对话 + 上下文记忆 + 对话历史 |
| 学习内容 | 纯文本讲解 | 结构化分段讲解 + 互动式例题 |
| 练习系统 | 选择题/填空题 | + 解题过程分步引导 + AI解析 |
| 诊断报告 | 静态展示 | 试卷拍照上传 → AI分析 → 生成计划 |
| 内容库 | 无 | 左滑呼出知识点浏览（对标白板草图） |
| UI品质 | 能看 | 动效完整，接近产品级 |
| 测试覆盖 | 手动测试 | 核心接口自动化测试 |
| 演示准备 | 1个下午 | 2天打磨 + 彩排 |

## 技术选型

| 层级 | 技术 | 理由 |
|------|------|------|
| 前端 | React 18 + TypeScript + Tailwind CSS | 组件化快速开发，Tailwind契合极简风格 |
| 前端UI库 | Headless UI + Framer Motion | 无样式约束 + 流畅动效 |
| 后端 | Python FastAPI | 异步高性能，AI集成天然友好 |
| 数据库 | PostgreSQL 15 | 设计文档指定，JSONB支持好 |
| 缓存 | Redis | 会话管理 + 对话上下文缓存 |
| AI | 通义千问/GPT-4 API | 直接调用，不自训练 |
| 公式渲染 | KaTeX | 比MathJax快10倍，适合实时渲染 |
| 图表 | Recharts | 轻量，React原生，用于诊断报告 |
| 部署 | Docker Compose | 一键启动全栈环境 |

## Demo演示范围

### ✅ 包含
- 用户登录 + 画像设置（年级/教材/目标院校）
- 极简主界面（目标倒计时 + 当前任务 + 艾乐入口）
- 内容库侧滑面板（知识点树形浏览）
- AI日清对话（多轮上下文、意图识别、任务生成）
- 学习计划自动生成与管理
- 学习任务列表与状态流转
- 结构化知识点讲解（分段 + LaTeX + 互动例题）
- 练习题系统（选择题/填空题 + AI个性化解析）
- 试卷诊断（图片上传 → AI分析 → 报告 → 生成计划）
- 知识掌握度可视化

### ❌ 不包含
- 数字分身/语音交互/视频课
- 多端切换（仅Web端）
- 文具库侧滑面板
- 行为事件埋点分析系统
- 向量数据库/语义检索

## Day 2（周二）：后端核心CRUD接口全量开发

### 上午（4h）
- [ ] 用户模块接口
  - `POST /api/auth/login` — 手机号+验证码登录（Mock固定验证码"8888"）
  - `POST /api/auth/refresh` — Token刷新
  - `GET /api/users/me` — 获取当前用户完整信息
  - `PUT /api/users/me` — 更新画像（年级、教材版本、目标院校、昵称、头像）
  - JWT签发/验证中间件 + 统一异常处理
- [ ] 学习计划模块接口
  - `GET /api/plans` — 用户计划列表（支持status筛选+分页）
  - `POST /api/plans` — 创建计划（标题、关联知识点）
  - `GET /api/plans/{id}` — 计划详情（含任务列表统计）
  - `PUT /api/plans/{id}` — 更新计划（标题、状态）
  - `DELETE /api/plans/{id}` — 归档计划

### 下午（4h）
- [ ] 学习任务模块接口
  - `GET /api/plans/{plan_id}/tasks` — 计划下的任务列表（按status分组）
  - `GET /api/tasks/{id}` — 任务详情（含知识点信息、内容包、关联练习题）
  - `PUT /api/tasks/{id}/status` — 状态流转（pending→in_progress→completed/skipped）
  - `POST /api/tasks/{id}/start` — 开始任务（记录started_at）
  - `POST /api/tasks/{id}/complete` — 完成任务（记录completed_at，更新计划进度）
- [ ] 知识点 + 题库模块接口
  - `GET /api/knowledge-points` — 知识点列表（支持subject筛选、树形结构返回）
  - `GET /api/knowledge-points/{id}` — 知识点详情（含先修关系链）
  - `GET /api/knowledge-points/{id}/exercises` — 知识点关联的练习题
  - `GET /api/exercises/{id}` — 题目详情
  - `POST /api/exercises/{id}/check` — 提交答案，返回判定+解析
- [ ] 编写完整的Pydantic请求/响应Schema
- [ ] Swagger文档自动生成，验证所有接口

### Day 2 交付标准
> Swagger UI可交互测试所有接口，CRUD逻辑正确，JWT鉴权生效，响应格式统一。

## Day 3（周三）：AI对话引擎 + 日清旅程完整实现

### 上午（4h）
- [ ] 封装大模型调用层 `LLMService`
  - 支持通义千问 / OpenAI双通道，配置切换
  - 流式输出（SSE）封装
  - Token用量统计与限流
  - 超时处理（10秒兜底）+ 自动重试（最多2次）
  - 错误降级：API不可用时返回预设回复
- [ ] 设计Prompt体系（3层结构）
  - **System Prompt**：艾乐人设（温暖学伴、鼓励式、高中数学专家）
  - **Context Prompt**：用户画像 + 当前学习计划 + 最近任务状态
  - **Task Prompt**：针对不同意图的专用模板（概念讲解/解题辅导/计划建议）
- [ ] Redis对话上下文管理
  - 存储最近20轮对话历史
  - 会话过期策略（24小时）
  - 上下文窗口滑动（防止Token超限）

### 下午（4h）
- [ ] 日清对话接口
  - `POST /api/chat/send` — 发送消息（SSE流式返回AI回复）
  - `GET /api/chat/sessions` — 会话列表（按日期分组）
  - `GET /api/chat/sessions/{session_id}/messages` — 会话消息历史
  - `POST /api/chat/sessions` — 创建新会话
- [ ] 意图识别与自动化流程
  - 调用大模型做意图分类，输出结构化JSON：
    ```json
    {
      "intent": "CLARIFY_CONCEPT",
      "confidence": 0.92,
      "slots": {
        "knowledge_point": "正弦函数定义",
        "knowledge_point_id": "kp_trig_sin"
      }
    }
    ```
  - 意图→动作映射：
    - `CLARIFY_CONCEPT` → 创建concept_learning任务 + 生成讲解内容
    - `SOLVE_PROBLEM` → 创建practice任务 + 推荐相关练习题
    - `PLAN_REQUEST` → 调用计划生成服务
    - `GENERAL_CHAT` → 纯对话回复
  - 自动写入 `daily_problems` 表
  - 自动创建 `learning_tasks` 并关联到当前计划
- [ ] 测试完整对话链路：用户说"三角函数没听懂" → 意图识别 → 任务创建 → AI回复确认

### Day 3 交付标准
> API层面完整跑通：对话→意图识别→任务自动生成→对话中确认，Redis上下文正常工作。

## Day 4（周四）：AI内容生成 + 诊断报告后端

### 上午（4h）
- [ ] AI内容生成服务
  - `POST /api/content/generate` — 根据知识点+任务类型生成结构化内容包
  - 内容包结构设计：
    ```json
    {
      "id": "cp_xxx",
      "sections": [
        { "type": "concept", "title": "什么是正弦函数", "content": "..." },
        { "type": "formula", "title": "核心公式", "content": "sin(θ) = \\frac{对边}{斜边}" },
        { "type": "example", "title": "典型例题", "content": "...", "interactive": true },
        { "type": "summary", "title": "本节小结", "content": "..." }
      ],
      "estimated_minutes": 15
    }
    ```
  - 生成后写入 `content_packages` 表，状态 generating → ready
  - 异步生成（后台任务），前端轮询状态
- [ ] AI个性化解题解析
  - `POST /api/exercises/{id}/ai-explain` — 针对用户的错误答案生成个性化解析
  - 不是固定解析，而是根据用户选的错误选项分析"你可能混淆了..."

### 下午（4h）
- [ ] 诊断报告模块
  - `POST /api/diagnosis/upload` — 上传试卷图片（存储到本地/OSS）
  - `POST /api/diagnosis/analyze` — 调用大模型分析试卷图片
    - 识别题目和用户答案
    - 逐题判定对错
    - 关联知识点
    - 生成知识掌握度摘要
  - `GET /api/diagnosis/reports` — 用户的诊断报告列表
  - `GET /api/diagnosis/reports/{id}` — 报告详情
  - `POST /api/diagnosis/reports/{id}/generate-plan` — 基于报告生成巩固学习计划
- [ ] 学习计划AI生成服务
  - 输入：薄弱知识点列表 + 用户画像
  - 输出：结构化的学习计划（含任务列表、预估时长、优先级排序）
  - 自动创建 `learning_plans` + 批量创建 `learning_tasks`

### Day 4 交付标准
> 内容生成、个性化解析、试卷诊断、计划自动生成四个AI功能接口全部可用。
