# 艾乐学伴 MVP Demo — 7天冲刺工作计划

## 总体目标

在7天内交付一个可流畅演示的艾乐学伴MVP Demo，覆盖核心学习闭环：
**用户登录 → 设定目标 → AI日清对话 → 生成学习计划/任务 → 知识点讲解 → 练习做题 → 完成反馈**

## 技术选型（推荐）

| 层级　　 | 技术　　　　　　　　　　　　　　　　 | 理由　　　　　　　　　　　　　　　　　　　 |
| ----------| --------------------------------------| --------------------------------------------|
| 前端　　 | React 18 + TypeScript + Tailwind CSS | 组件化开发快，Tailwind极简风格契合产品理念 |
| 后端　　 | Python FastAPI　　　　　　　　　　　 | 异步高性能，与AI模型集成天然友好　　　　　 |
| 数据库　 | PostgreSQL 15　　　　　　　　　　　　| 设计文档指定，JSONB支持好　　　　　　　　　|
| AI　　　 | 通义千问 API　　　　　　　　　　　　 | 直接调用，不自训练　　　　　　　　　　　　 |
| 部署　　 | Docker Compose　　　　　　　　　　　 | 一键启动全栈环境　　　　　　　　　　　　　 |
| 状态管理 | Zustand　　　　　　　　　　　　　　　| 轻量，适合MVP　　　　　　　　　　　　　　　|

## Demo演示范围（明确边界）

### ✅ 包含（核心闭环）
- 用户登录（手机号简化版）
- 用户画像设置（年级、教材版本、目标院校）
- 极简主界面（目标倒计时 + 当前任务卡片 + 艾乐入口）
- AI日清对话（文字交互，意图识别，问题澄清）
- 学习计划自动生成
- 学习任务列表与状态流转
- 知识点AI讲解（大模型实时生成文本）
- 练习题交互（选择题/填空题，提交判定，显示解析）
- 诊断报告（基础版）

### ❌ 不包含（后续迭代）
- 数字分身/语音交互/视频课
- 多端切换（仅做Web端）
- 内容库/文具库侧滑面板
- 行为事件埋点分析
- 向量数据库/知识图谱语义检索
- 真实短信验证码

## Day 2（周二）：用户系统 + 后端核心CRUD

### 上午（3h）
- [ ] 后端：用户模块
  - `POST /api/auth/login` — 手机号登录（Mock验证码，固定"8888"通过）
  - `GET /api/users/me` — 获取当前用户信息
  - `PUT /api/users/me` — 更新用户画像（年级、教材版本、目标院校）
  - JWT Token签发与验证中间件
- [ ] 后端：学习计划模块
  - `GET /api/plans` — 获取用户所有计划
  - `POST /api/plans` — 创建学习计划
  - `GET /api/plans/{id}` — 获取计划详情（含任务列表）
  - `PUT /api/plans/{id}/status` — 更新计划状态

### 下午（3h）
- [ ] 后端：学习任务模块
  - `GET /api/tasks` — 获取当前计划的任务列表（支持status筛选）
  - `GET /api/tasks/{id}` — 获取任务详情（含知识点、内容包、练习题）
  - `PUT /api/tasks/{id}/status` — 更新任务状态（pending→in_progress→completed）
  - `POST /api/tasks/{id}/submit-answer` — 提交练习题答案，返回判定结果
- [ ] 后端：知识点 + 题库查询
  - `GET /api/knowledge-points` — 知识点列表（支持subject筛选）
  - `GET /api/knowledge-points/{id}` — 知识点详情（含先修关系）
  - `GET /api/exercises` — 按知识点查询练习题
- [ ] 编写Pydantic响应Schema，确保前后端数据契约清晰

### Day 2 交付标准
> 用Postman/Swagger跑通所有接口，数据正确返回，JWT鉴权生效。

## Day 3（周三）：AI对话引擎 + 日清旅程后端

### 上午（3h）
- [ ] 封装大模型调用层（支持通义千问/OpenAI，可切换）
  - 统一的 `LLMService` 类
  - 流式输出（SSE）支持
  - Prompt模板管理（System Prompt定义艾乐学伴人设）
  - 错误处理与重试机制
- [ ] 设计艾乐学伴的System Prompt：
  ```
  你是"艾乐"，一个温暖、耐心的AI学习伙伴（不是老师）。
  你的任务是帮助高中生理解数学知识、解决学习困惑。
  你说话风格亲切、鼓励式，像一个学霸同桌。
  当学生提出问题时，你需要：
  1. 判断问题意图（概念不清/解题困难/学习规划）
  2. 用简洁易懂的方式解答
  3. 适时推荐练习题巩固
  ```

### 下午（3h）
- [ ] 后端：日清对话模块
  - `POST /api/chat/message` — 发送消息，返回AI回复（SSE流式）
  - `GET /api/chat/sessions` — 获取历史会话列表
  - `GET /api/chat/sessions/{id}/messages` — 获取会话消息记录
- [ ] 后端：意图识别与任务生成
  - 调用大模型做意图分类（CLARIFY_CONCEPT / SOLVE_PROBLEM / PLAN_REQUEST）
  - 从对话中提取知识点槽位（knowledge_point_ids）
  - 自动创建 `daily_problems` 记录
  - 根据意图自动生成 `learning_tasks`（关联到当前计划）
- [ ] 后端：AI内容生成
  - `POST /api/content/generate` — 根据知识点生成讲解内容包
  - 内容包结构：`{ sections: [{ type: "text"|"example", content: "..." }] }`

### Day 3 交付标准
> 通过API能和艾乐对话，AI能识别"我不懂三角函数"这类问题，自动创建学习任务。

## 风险与应对

| 风险　　　　　　　　　　| 概率 | 应对策略　　　　　　　　　　　　　　　　　　　 |
| -------------------------| ------| ------------------------------------------------|
| AI API调用不稳定/延迟高 | 中　 | 准备Mock回复数据，演示时可切换为预录模式　　　 |
| 前后端联调耗时超预期　　| 高　 | Day5整天留给联调，提前定义好API Schema　　　　 |
| Docker环境差异　　　　　| 低　 | 锁定所有依赖版本，提前在目标机器测试　　　　　 |
| 种子数据不够真实　　　　| 中　 | Day6专门准备，找真实高中数学题填充　　　　　　 |

## 大模型辅助编码策略

- Day1-2：让大模型批量生成ORM模型、CRUD接口、Pydantic Schema（效率最高的部分）
- Day3：让大模型帮写Prompt模板和意图识别的few-shot examples
- Day4-5：让大模型生成React组件骨架，手动调整样式和交互细节
- Day6：让大模型生成演示用的种子数据（知识点、题目、对话记录）
- 全程：遇到Bug先让大模型分析，节省调试时间
